CIRCL¶

Version: main branch snapshot

Repository: https://github.com/cloudflare/circl

Docs:

Primitives¶

X25519, X448

X25519¶

KeyGen:

  • Montgomery
  • Right-to-left Joye ladder via KeyGen -> ladderJoye
  • xz
  • Scalar clamping via k[0] &= 248 and k[31] = (k[31] & 127) | 64
  • Fixed-point multiplication with the generator point

Derive:

  • Montgomery
  • Left-to-right Montgomery ladder via Shared -> ladderMontgomery
  • xz
  • Same scalar clamping as in KeyGen
  • Public-key validation against low-order points inside Shared

Implementation notes:

  • ladderJoye is documented in source as a fixed-point multiplication using the generator.
  • ladderMontgomery is documented in source as a generic scalar point multiplication.
  • Internal ladder state uses 5 field elements: [x1, x2, z2, x3, z3] for Montgomery multiplication.

X448¶

KeyGen:

  • Montgomery
  • Right-to-left Joye ladder via KeyGen -> ladderJoye
  • xz
  • Scalar clamping via k[0] &= 252 and k[55] |= 128
  • Fixed-point multiplication with the generator point

Derive:

  • Montgomery
  • Left-to-right Montgomery ladder via Shared -> ladderMontgomery
  • xz
  • Same scalar clamping as in KeyGen
  • Public-key validation against low-order points inside Shared

Implementation notes:

  • The package implements Diffie-Hellman as specified in RFC 7748.
  • Source files include generic and amd64-specific code paths, including curve_generic.go, curve_amd64.go, and curve_amd64.s.
  • The ladder code uses Montgomery differential-addition / doubling structure over fp448.

Observations¶

  • CIRCL uses different ladder variants for fixed-point key generation and variable-point shared-secret derivation.
  • Both X25519 and X448 combine scalar clamping, xz coordinates, and protocol-level low-order public-key validation.
  • The implementation is package-specific rather than a single unified ECC engine.
